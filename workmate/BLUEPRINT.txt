1. PROJECT OVERZICHT
================================================================================

  Naam:           Workmate
  Type:           Single Page Application (SPA)
  Doel:           Alles-in-een bedrijfsmanagement tool voor signbedrijven
  Stack:          React 18 + TypeScript + Vite + Tailwind CSS + Supabase
  Deployment:     Vercel (serverless)
  Taal interface: Nederlands (primair) + Engels

  Modules:        17 functionele modules + 37 componentmappen
  Routes:         46 (39 beschermd + 7 publiek)
  Database:       30+ tabellen gedefinieerd (Supabase/PostgreSQL)

6. DATA-LAAG: DUAL-MODE ARCHITECTUUR
================================================================================

  De app heeft een uniek "Supabase-first, localStorage-fallback" systeem:

  ┌─────────────────────────────────────────────────┐
  │  supabaseService.ts — Elke CRUD functie:        │
  │                                                  │
  │  if (isSupabaseConfigured() && supabase) {       │
  │    → Echte Supabase query                        │
  │  } else {                                        │
  │    → localStorage fallback                       │
  │  }                                               │
  └─────────────────────────────────────────────────┘

  HUIDIGE MODUS:    localStorage (demo)
  REDEN:            Geen .env bestand met Supabase credentials
  OMSCHAKELEN:      .env aanmaken met VITE_SUPABASE_URL + VITE_SUPABASE_ANON_KEY

  Demo mode gedrag:
  - Bij eerste bezoek worden voorbeelddata geladen via useDataInit.ts
  - 5 voorbeeldklanten, 3 projecten, 4 offertes, 5 taken
  - Auth werkt met een "demo-user" in localStorage
  - Data verdwijnt bij browser cache wissen

8. HUIDIGE PROBLEMEN
================================================================================

  KRITIEK (blokkeert deployment)
  ------------------------------

  K1. TypeScript Build Errors (~28 fouten)
      - useDataInit.ts: seed data heeft verkeerde veldnamen
      - Offerte type mist 'beschrijving' veld
      - Taak status "open" bestaat niet (moet "todo"|"bezig"|"review"|"klaar")
      - MontageAfspraak mist velden (materialen, notities)
      - Diverse type mismatches in demo data
      Impact: `npm run build` faalt, deployment onmogelijk

  K2. Geen actieve database
      - Supabase niet geconfigureerd
      - Alle data in localStorage (verdwijnt bij cache wissen)
      - Geen multi-user ondersteuning
      - Geen data persistentie tussen apparaten

  HOOG (blokkeert productie)
  --------------------------

  H1. Geen tests
      - 0 test bestanden
      - Geen test framework geconfigureerd
      - Geen test scripts in package.json

  H2. Geen CI/CD pipeline
      - Geen GitHub Actions
      - Geen automatische builds of checks
      - Handmatige deployment enige optie

  H3. Demo modus onveilig
      - Auth tokens in localStorage als plaintext
      - Geen sessie expiratie
      - "demo-user" ID voorspelbaar

  MEDIUM (voor productie-kwaliteit)
  ---------------------------------

  M1. CORS te breed
      - Access-Control-Allow-Origin: * in vercel.json
      - Moet beperkt worden tot eigen domein

  M2. Geen PWA ondersteuning
      - Geen manifest.json
      - Geen service worker
      - Niet installeerbaar op mobiel

  M3. Geen error monitoring
      - Geen Sentry of vergelijkbaar
      - Fouten verdwijnen onopgemerkt in productie

  M4. Geen Content Security Policy
      - Geen CSP headers geconfigureerd

  M5. i18n beperkingen
      - Vertalingen hardcoded in LanguageContext (~29K regels)
      - Geen dynamische loading
      - Geen pluralisatie
